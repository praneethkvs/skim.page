# skim.page

`skim.page` is a lightweight URL-prepend app for turning long articles into AI-ready TL;DR prompts.

The Phase 1 product is client-only: a user prepends `skim.page/` to an article URL, optionally adds a provider and lens, then the app generates a prompt, attempts to open the selected provider automatically, and shows a copyable fallback prompt.

## Current Scope

Phase 1 includes:

- Minimal Vite + React + TypeScript app.
- Landing page based on the existing `skim.page.html` visual direction.
- Four built-in lenses: Default, ELI5, Research, Investor.
- Five provider families: ChatGPT, Gemini, Claude, Perplexity, Grok.
- Path-style and query-param URL parsing.
- Generated prompt fallback page with `Open {Provider}` and `Copy prompt`.
- Malformed URL state.
- Client-only static hosting assumptions.

Phase 1 does not include:

- Backend services.
- OpenAI API keys or direct model calls.
- Article scraping or article content extraction.
- Accounts, database, analytics, or saved preferences.
- Browser extension implementation.
- Direct in-app summarization.

## URL Contract

Default provider is ChatGPT. Default lens is Default.

Supported path examples:

```text
/https://example.com/article
/i/https://example.com/article
/r/https://example.com/article
/e/https://example.com/article
/cl/r/https://example.com/article
/px/https://example.com/article
/ge/e/https://example.com/article
/grok/default/https://example.com/article
```

Supported query example:

```text
/?url=https%3A%2F%2Fexample.com%2Farticle&lens=investor&provider=cl
```

Normalization rules:

- Preserve `http://` and `https://`.
- Tolerate hosted redirects that collapse `https://` in path-style article URLs to `https:/`.
- Add `https://` to likely domain/path inputs without a protocol.
- Missing or unsupported lens falls back to Default.
- Missing or unsupported provider falls back to ChatGPT.
- Malformed article input shows a graceful error state.

## Provider Contract

Provider aliases:

```text
ch, chatgpt, openai -> ChatGPT
ge, gemini -> Gemini
cl, claude -> Claude
px, perplexity -> Perplexity
gr, grok -> Grok
```

Lens aliases:

```text
default, d -> Default
eli5, e -> ELI5
research, r -> Research
investor, i -> Investor
```

Current handoff URLs:

```text
ChatGPT:    https://chatgpt.com/?prompt={encodedPrompt}
Gemini:     https://www.google.com/search?udm=50&q={encodedPrompt}
Claude:     https://claude.ai/new?q={encodedPrompt}
Perplexity: https://www.perplexity.ai/?q={encodedPrompt}
Grok:       https://grok.com/?q={encodedPrompt}
```

Handoff behavior is best-effort. The fallback page must always remain available because third-party URL prefill behavior can change or the browser may block automatic pop-ups. The fallback page explains that users can allow pop-ups and redirects for skim.page once so future handoffs open automatically.

Gemini note: Gemini currently opens through Google AI Mode search using `udm=50`. This is useful but not a guaranteed public API. Future extension work may inject prompts directly into Gemini instead.

## Lens Prompts

The final generated prompt format is:

```text
{MODE_PROMPT}

Article URL: {ARTICLE_URL}
```

Default:

```text
Summarize this article briefly.

Provide:
- TLDR in 1-2 sentences
- 3 key takeaways
- Why it matters in 1 sentence
```

Investor:

```text
Analyze this article from an investor perspective.

Provide:
- TLDR in 1-2 sentences
- Bull case in 2 bullets
- Bear case in 2 bullets
- Business or market implication
- Key metric, claim, or risk to watch
```

Research:

```text
Analyze this article like a researcher.

Provide:
- TLDR in 1-2 sentences
- Main argument
- 3 key pieces of evidence or claims
- Any major assumptions or limitations
- One open question
```

ELI5:

```text
Explain this article in simple terms.

Provide:
- TLDR in 1-2 sentences
- Simple explanation in 3-5 bullets
- One real-world analogy
- Why it matters in 1 sentence
```

## UI Decisions

- Keep the product name as `skim.page`.
- Landing page uses hero content on the left and provider/lens controls on the right on medium/large screens.
- Header bottom border appears only after scroll.
- Removed the original stats block and `BETA` badge.
- Font sizes were increased from the initial port.
- Lens controls auto-cycle until the user interacts with lens/provider controls.
- Prompt preview height is fixed so auto-cycling lenses does not shift the layout.
- Hero prefix changes dynamically with selected provider and lens, using short lens aliases like `e`, `r`, and `i`.
- Hero prefix includes a `Copy` button so users can copy `https://skim.page/.../`.
- Hero eyebrow cycles with the selected lens: `READ SMARTER`, `READ BETTER`, `READ DEEPER`, `READ FASTER`.
- Provider label is `Choose Your Provider`; lens label is `Choose your Lens`.
- ChatGPT helper copy says it is the default.
- Provider display order is ChatGPT, Gemini, Claude, Perplexity, Grok.
- Lens display order is Default, ELI5, Research, Investor.
- Provider and lens buttons include shortcut hints, such as `Gemini (ge)` and `ELI5 (e)`.
- How It Works includes provider shortcut references: `ch`, `ge`, `cl`, `px`, `gr`.
- How It Works includes a compact note that shortcuts and full names both work.
- `index.html` includes the public title, description, favicon link, Open Graph, Twitter card, and theme-color metadata.
- `vercel.json` rewrites all routes to `index.html` so prepend-style paths load the SPA on Vercel.

## Recent Decisions

- Phase 1 now includes provider family selection, not individual model selection.
- Short provider aliases and full provider names both work.
- Single-letter lens aliases now work: `i`, `r`, and `e`.
- Full lens names still work anywhere a shortcut works.
- ChatGPT is shown in provider UI as the default; `ch` remains supported as a URL alias.
- ChatGPT remains the default provider.
- Gemini no longer links directly to `gemini.google.com/app` for the web handoff; it uses Google AI Mode search via `https://www.google.com/search?udm=50&q=...`.
- Gemini remains best-effort. A future browser extension can make Gemini more reliable by opening Gemini, filling the prompt editor, and optionally submitting.
- Browser extension, custom lenses, saved provider defaults, and model-specific routing are future work.

## Phase 2 Direction

Future browser extension features:

- Trigger `skim.page` from the current tab without manual address-bar editing.
- Pick built-in lenses.
- Create and save custom lenses with custom user prompts.
- Store extension-level defaults.
- Store optional provider/model preferences.
- For Gemini, use a content script to open `gemini.google.com/app`, inject the generated prompt into the prompt editor, optionally auto-submit, and preserve copy fallback if injection fails.

Future multi-model support:

- Phase 1 supports provider families only.
- Phase 2 may add `model` as a query parameter and saved preference.
- Example future URL:

```text
/?url=https%3A%2F%2Fexample.com%2Farticle&lens=investor&provider=openai&model=gpt-4.1
```
